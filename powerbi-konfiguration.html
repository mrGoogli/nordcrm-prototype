📊 Hvorfor DirectQuery og ikke Import? Med DirectQuery henter Power BI data direkte fra Synapse ved hvert rapportopkald — ingen forældet cache. Filtre (KAM, periode, stadie) slås igennem til databaseforespørgsler. Det er lidt langsommere (<3 sek.) men altid aktuelt. Import er hurtigere men kræver daglig refresh-planlægning.
🔗 Synapse Link for Cosmos DB Azure Synapse Link er en nøglefunktion: den synkroniserer automatisk al data fra Cosmos DB til Synapse Analytics Analytical Store (columnar format) uden ETL-pipelines. Latency: typisk under 2 minutter. Aktiveres med én indstilling på Cosmos DB-kontoen.

Provisionér Power BI Embedded-kapacitet og konfigurér Entra ID Service Principal

2.1 Deploy Bicep — Power BI Embedded kapacitet Azure CLI ▶
Power BI Embedded-kapaciteten er allerede defineret i modules/powerbi.bicep fra IaC-pakken. Kør bare main Bicep-deploymenten — kapaciteten oprettes automatisk i North Europe.
Bicep deploy (fra IaC-pakken)
# Log ind og sæt subscription az login az account set --subscription "<YOUR-SUBSCRIPTION-ID>" # Deploy hele NordCRM infrastruktur (inkl. Power BI Embedded) az deployment sub create \ --location northeurope \ --template-file main.bicep \ --parameters @main.bicepparam # Verificér kapaciteten az powerbi embedded-capacity show \ --name pbinordcrmoprod \ --resource-group rg-nordcrm-prod
💰 Pris: SKU A1 A1 koster ca. DKK 5.000-6.000/måned (kørende 24/7). Overvej at pause kapaciteten uden for arbejdstid via Azure Automation — kan spare 60% af omkostningerne. Skala til A2 (~10.000 kr/mdr.) hvis >10 samtidige brugere.
2.2 Opret App Registration (Service Principal) Azure Portal ▶

Service Principalen bruges af NordCRM backend til at generere embed-tokens uden at nogen bruger er logget ind.

CLI alternativ til portal
# Opret App Registration az ad app create \ --display-name "NordCRM-PowerBI-SP" \ --sign-in-audience AzureADMyOrg # Gem appId (clientId) APP_ID=$(az ad app list --display-name "NordCRM-PowerBI-SP" \ --query "[0].appId" -o tsv) # Opret service principal az ad sp create --id $APP_ID # Opret client secret (gem i Key Vault!) az ad app credential reset \ --id $APP_ID \ --years 2 \ --query "{clientId:appId, secret:password}" # Gem secret i Azure Key Vault az keyvault secret set \ --vault-name kv-nordcrm-prod \ --name pbi-sp-client-secret \ --value "<SECRET-FROM-ABOVE>"

Derefter: I Power BI Admin Portal → Tenant Settings → aktivér "Service principals can use Power BI APIs" og tilføj security group med service principalen.

2.3 Aktivér Synapse Link på Cosmos DB Azure CLI ▶
Synapse Link er den "magiske" forbindelse. Aktivering tager 5-10 minutter. Derefter synkroniseres alle Cosmos DB-containere automatisk til Synapse Analytical Store.
Aktivér Synapse Link
# Aktivér Synapse Link på Cosmos DB-kontoen az cosmosdb update \ --name cosmos-nordcrm-prod \ --resource-group rg-nordcrm-prod \ --enable-analytical-storage true # Aktivér analytical store på containers der skal bruges i Power BI # (deals, contacts, accounts, users, activities, audit_log) for container in deals contacts accounts users activities; do az cosmosdb sql container update \ --account-name cosmos-nordcrm-prod \ --database-name nordcrm \ --name $container \ --resource-group rg-nordcrm-prod \ --analytical-storage-ttl -1 done # Link Synapse workspace til Cosmos DB # (Gøres i Synapse Studio: Data → + → Connect to external data → Azure Cosmos DB)
03 / 07
Synapse Analytics — SQL Views
Definer de SQL-views i Synapse der udgør datamodellen for Power BI
⏱ 45 min
📐 Datamodel-strategi Vi bruger Synapse Serverless SQL Pool med OPENROWSET til at læse direkte fra Cosmos DB Analytical Store. Views transformerer de flade JSON-dokumenter til tabelformat som Power BI forstår. Dette kræver ingen separat ETL-pipeline.
3.1 View: vw_deals — aftaler med AM/KAM T-SQL ▶
Synapse Serverless SQL — Kør i Synapse Studio
-- Opret database til NordCRM views CREATE DATABASE nordcrm_reporting; USE nordcrm_reporting; GO -- View: deals med AM, KAM, kontakt og konto-data CREATE OR ALTER VIEW vw_deals AS SELECT d.id AS deal_id, d.tenantId, d.title, d.stage, d.valueOere / 100.0 AS value_dkk, -- øre → DKK d.currency, d.probability, d.aiProbability AS ai_probability, d.expectedCloseDate, d.closedAt, d.createdAt, d.updatedAt, -- Kontaktinfo c.firstName + ' ' + c.lastName AS contact_name, c.email AS contact_email, -- Kontoinfo a.name AS account_name, a.cvrNumber AS account_cvr, a.industry, -- Flåde data (fra accounts.fleetData) a.fleetData.fleetSize AS fleet_size, a.fleetData.opendoFleet AS opendo_fleet, CASE WHEN a.fleetData.fleetSize > 0 THEN CAST(a.fleetData.opendoFleet AS FLOAT) / CAST(a.fleetData.fleetSize AS FLOAT) * 100 ELSE 0 END AS sow_pct, -- AM / KAM (fra accounts) a.accountManager.name AS am_name, a.accountManager.userId AS am_user_id, a.keyAccountManager.name AS kam_name, a.keyAccountManager.userId AS kam_user_id, -- Beregnet: vundet/tabt CASE d.stage WHEN 'won' THEN 1 WHEN 'lost' THEN 0 ELSE NULL END AS is_won FROM OPENROWSET('CosmosDB', 'Account=cosmos-nordcrm-prod;Database=nordcrm;Key=@key', deals) WITH ( id VARCHAR(64), tenantId VARCHAR(64), title NVARCHAR(500), stage VARCHAR(50), valueOere BIGINT, currency VARCHAR(3), probability INT, aiProbability FLOAT, contactId VARCHAR(64), accountId VARCHAR(64), expectedCloseDate DATE, closedAt DATETIME2, createdAt DATETIME2, updatedAt DATETIME2 ) AS d LEFT JOIN OPENROWSET('CosmosDB', ..., contacts) c ON d.contactId = c.id AND d.tenantId = c.tenantId LEFT JOIN OPENROWSET('CosmosDB', ..., accounts) a ON d.accountId = a.id AND d.tenantId = a.tenantId;
3.2 DAX-målinger til Power BI DAX ▶
Power BI Desktop — DAX Measures
// ── PIPELINE MEASURES ── Pipeline Total DKK = SUMX( FILTER(vw_deals, vw_deals[stage] <> "won" && vw_deals[stage] <> "lost"), vw_deals[value_dkk] ) Pipeline Vægtet DKK = SUMX( FILTER(vw_deals, vw_deals[stage] <> "won"), vw_deals[value_dkk] * vw_deals[probability] / 100 ) Win Rate % = DIVIDE( COUNTROWS(FILTER(vw_deals, vw_deals[is_won] = 1)), COUNTROWS(FILTER(vw_deals, vw_deals[is_won] <> BLANK())), 0 ) Vundne DKK = CALCULATE(SUM(vw_deals[value_dkk]), vw_deals[stage] = "won") // ── SOW MEASURES ── SOW % Gennemsnit = AVERAGEX(FILTER(vw_deals, vw_deals[fleet_size] > 0), vw_deals[sow_pct]) SOW Farve = VAR sow = [SOW % Gennemsnit] RETURN IF(sow >= 25, "#1a6b4a", -- Grøn: >=25% IF(sow >= 10, "#9a4f00", -- Amber: 10-25% "#b91c1c")) -- Rød: <10% Total Flåde = SUM(vw_deals[fleet_size]) Opendo Flåde = SUM(vw_deals[opendo_fleet]) // ── DAGE TIL LUKNING ── Dage til lukning (gns.) = AVERAGEX( FILTER(vw_deals, vw_deals[stage] <> "won" && vw_deals[stage] <> "lost"), DATEDIFF(TODAY(), vw_deals[expectedCloseDate], DAY) )

5.1 Backend — Embed Token Service (Node.js / TypeScript) TypeScript ▶
src/services/powerbi-embed.service.ts
import { ConfidentialClientApplication } from '@azure/msal-node'; import axios from 'axios'; const PBI_SCOPE = 'https://analysis.windows.net/powerbi/api/.default'; const PBI_API = 'https://api.powerbi.com/v1.0/myorg'; interface EmbedToken { token: string; tokenId: string; expiration: string; embedUrl: string; reportId: string; } export class PowerBIEmbedService { private msalClient: ConfidentialClientApplication; constructor() { this.msalClient = new ConfidentialClientApplication({ auth: { clientId: process.env.PBI_SP_CLIENT_ID!, clientSecret: process.env.PBI_SP_CLIENT_SECRET!, // Fra Key Vault authority: `https://login.microsoftonline.com/${process.env.AZURE_TENANT_ID}`, } }); } private async getAccessToken(): Promise<string> { const result = await this.msalClient.acquireTokenByClientCredential({ scopes: [PBI_SCOPE] }); if (!result?.accessToken) throw new Error('PBI token acquisition failed'); return result.accessToken; } /** * Generér embed-token med RLS-roller for den aktuelle bruger. * Filtre sendes fra NordCRM frontend (kam_id, am_id, period, stage). */ async generateEmbedToken( workspaceId: string, reportId: string, userRole: 'am' | 'kam' | 'admin', userId: string, tenantId: string ): Promise<EmbedToken> { const accessToken = await this.getAccessToken(); // Hent embed-URL til rapporten const reportRes = await axios.get( `${PBI_API}/groups/${workspaceId}/reports/${reportId}`, { headers: { Authorization: `Bearer ${accessToken}` } } ); const embedUrl = reportRes.data.embedUrl; // Generer embed-token med RLS const tokenRes = await axios.post( `${PBI_API}/groups/${workspaceId}/reports/${reportId}/GenerateToken`, { accessLevel: 'view', // RLS: sæt brugerens rolle og identitet identities: [{ username: userId, roles: [userRole], // 'am', 'kam' eller 'admin' datasets: [reportRes.data.datasetId] }] }, { headers: { Authorization: `Bearer ${accessToken}` } } ); // Audit log: PBI embed access (GDPR — Art. 30) await auditLog({ action: 'pbi_embed_access', userId, tenantId, reportId }); return { ...tokenRes.data, embedUrl, reportId }; } }

Power BI Desktop — RLS roller (Modelling → Manage Roles)
-- Rolle: AM — ser kun egne aftaler -- Tabel: vw_deals — DAX-filter [am_user_id] = USERPRINCIPALNAME() -- Rolle: KAM — ser alle aftaler under sit team -- Kræver en 'user_hierarchy' tabel: kam_id → [am_id, am_id, ...] [kam_user_id] = USERPRINCIPALNAME() || [am_user_id] IN VALUES(FILTER(user_hierarchy, user_hierarchy[kam_id] = USERPRINCIPALNAME())[am_id]) -- Rolle: Admin — ingen filter (blank DAX = ser alt) -- (Tom fil i Manage Roles → admin-rollen) -- Testér RLS i Power BI Desktop: -- Modelling → View as Roles → vælg 'am' → indtast test-brugernavn
✓ GDPR Art. 5 — Dataminimering via RLS RLS er den tekniske implementering af dataminimering i Power BI. En AM har ikke adgang til andre AM'ers kundedata, selvom de forsøger at fjerne slicere eller eksportere. RLS håndhæves server-side i Power BI Premium-kapaciteten — ikke kun i UI'en.
